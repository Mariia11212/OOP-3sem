# Лабораторна робота №X: Об'єктно-орієнтовані структури даних та геометрія на площині

## Виконаний функціонал

### 1. Вимоги ООП (демонстрація належної архітектури)
- **18 класів/типів** (значно більше мінімальних 9), що дозволяє створити чіткі абстракції для геометрії та структур даних.
- **Понад 20 полів** (більше 15), що забезпечує достатню деталізацію для об'єктів.
- **Понад 50 нетривіальних методів** (значно більше 25), які реалізують всю логіку обробки фігур, перетворень та операцій зі списками.
- **3 ієрархії успадкування** (більше 2, кожна з ≥ 3 класів):
  1. Геометричні фігури: `IShape -> Shape -> конкретні фігури`
  2. Перетворення: `ITransformation -> конкретні перетворення`
  3. Структури даних: `IDataList -> конкретні реалізації списків`
- **4 незалежні випадки поліморфізму** (більше 3), включаючи:
  - Динамічний для фігур, перетворень та списків
  - Статичний через Generics (`TransformationManager`)
- **Правильна інкапсуляція**: приватні поля, властивості з `private set`.
- **Активне використання абстрактних класів та інтерфейсів** (`Shape`, `IShape`, `ITransformation`, `IDataList`) для визначення архітектури та контрактів.

---

### 2. Розширений функціонал Геометрії на площині
- **Багатокутники (`Polygon`)** як узагальнена фігура, з якої успадковується `Rectangle`.
- **Три типи перетворень**: `Translation` (переміщення), `Rotation` (обертання), `Scaling` (масштабування).
- **Обчислення ключових властивостей** для всіх фігур: площа, периметр, центроїд/центр мас.
- **Додаткові алгоритми**:
  - Обчислення відстані між точками
  - Перевірка входження точки в коло
  - Знаходження середини відрізка

---

### 3. Розширений функціонал структур даних
- **Реалізовано три типи власних списків**:
  1. `MyDynamicArrayBasedList` (на основі масивів)
  2. `MyLinkedList` (зв'язний список)
  3. `MyCircularList` (циклічний список)
- Кожен список підтримує повний набір операцій:
  - Додавання, вставка, видалення
  - Пошук за значенням та за індексом
  - Розширений пошук першого елемента за заданою умовою (`FindFirst`)

---

### 4. Інтеграція та практична демонстрація
- **Колекції геометричних фігур** (`IShape`, `Rectangle`, `Circle`) в `Program.cs` зберігаються та обробляються з використанням власних структур даних (`MyDynamicArrayBasedList<T>`).  
- Це наочно демонструє **взаємодію геометрії та розроблених структур даних** і користь від компонентів.
